# Project Retrospective — Fishing Fundamentals: Knots & Baiting

**Author:** Christian Andrews  
**Date:** 03/01/2026

## What went well
- Clear scope and simple 2‑page structure (index.html + about.html) kept the project focused and easy to finish.  
- Using a single external stylesheet (style.css) ensured consistent styling across pages and simplified tweaks.  
- GitHub Desktop + GitHub Pages workflow allowed quick publishing and live verification.  
- Added a visible Live Site link in README.md so graders can find the site immediately.  
- Small, focused commits after the automatic initial commit produced a clean, readable history.

## What was challenging
- External resource certificate error (takemefishing.org) required finding an alternate solution and updating links.  
- Locating and optimizing images with correct attribution took more time than expected.  
- Ensuring mobile/responsive layout required iterative CSS adjustments and testing.  
- Getting comfortable with the Pages deployment delay (changes sometimes took 30–90s to appear) made immediate verification slower.

## What I would improve next time
- Start gathering and optimizing images earlier in the process to avoid last‑minute work.  
- Add a printable one‑page “field card” summary for quick reference while fishing.  
- Include a small glossary of fishing terms for beginners.  
- Create a short local testing checklist (broken links, image paths, accessibility checks) to run before publishing.  
- If needing micro‑commits for grading, initialize an empty repo or use the Git CLI to control the very first commit.

## Audience feedback (if any)
- Who tested it:

- What they struggled with:  
  - One tester noted the troubleshooting section could be more visible and suggested a short “Quick Start” block near the top.

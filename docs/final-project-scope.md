## 1. Project Purpose / Justification
Create a simple, instructional two-page static website to teach beginner fishermen how to tie essential fishing knots and how to bait hooks safely and clearly.
## 2. Project Objectives (SMART)
- Deliver a responsive two-page website (index.html, about.html) using one external stylesheet (style.css) in a public GitHub repo by the course deadline.
- Provide at least 5 knot tutorials and 3 baiting techniques with step-by-step text and at least one image per tutorial.
- Include docs/final-project-scope.md, docs/plan.md, docs/retrospective.md in the repo.
## 3. Deliverables
- index.html — homepage with tutorials (5 knots, 3 baiting techniques)
- about.html — about, author, sources, contact
- style.css — external stylesheet applied to both pages
- /assets or /images — tutorial images or diagrams
- README.md with live site link
- docs/final-project-scope.md (this file), docs/plan.md, docs/retrospective.md
## 4. Requirements
- Semantic HTML, proper heading structure, descriptive alt text for images.
- Readable fonts and adequate color contrast.
- No backend; minimal or no JavaScript.
- All styling via external CSS.
- Use royalty-free or self-created images with attribution.
## 5. Exclusions
- No video tutorials, interactive knot animations, user accounts, e-commerce, or backend services.
## 6. Acceptance Criteria
- Repository contains required files and docs in specified structure.
- Homepage includes 5 distinct knots and 3 baiting techniques with steps and images.
- index.html and about.html link to each other; style.css is external and applied.
- Site renders correctly on desktop and mobile; no broken links.
## 7. Constraints
- Time: course schedule and deadlines.
- Tools: static site (HTML/CSS), GitHub hosting (free tier).
- Budget: $0 (free tools only).
## 8. Assumptions
- Instructor accepts GitHub Pages or similar static hosting for the live link.
- Images will be available or created and properly attributed.
- Peer feedback may require minor revisions.
## 9. High-Level Risks & Mitigations
- Content errors — verify tutorials against two reputable sources.
- Poor or missing images — create simple diagrams or use royalty-free images.
- Time overrun — prioritize core content (5 knots, 3 techniques) and schedule tasks.
